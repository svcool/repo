#Регистрация на конкретный сервер
1.Кластеры->sip1->Транспорт-> Добавить IPset(kms) с одним адрессом и другим портом(5062)
2.Домены->свойства домена->SIP транспорт-> Добавить созданные в п1. IPsetы
3.На телефонах регистроваться на одни IP c портом 5062, c указанием домена.
4.Менеджер транков -> IP транк создавать на IPset(kms) на порт приема 5062. На SMG порт назначения сигнализации 5062. Указать домен.

Абонент на ECSS необходимо указывать оба адресса кластера, иначе будет периодическая недоступность выхода через транк.

#SSL сертификат
https://docs.eltex-co.ru/ecss31412/nastrojka-sertifikatov-ecss-10-381519802.html
В ECSS-10 генерируется и используется единый сертификат. Все подсистемы используют по умолчанию ecss10.crt.
Также в ходе установки пакета css-user генерируются следующие файлы:

ecss10root.key — корневой приватный ключ под паролем;
ecss10root.crt — корневой сертификат. Этот сертификат устанавливается на клиентский ПК, чтобы исключить предупреждения безопасности в браузере, если используется само подписанный сертификат, который генерирует ecss-user при установке;
ecss10.key — приватный ключ для использования системой;
ecss10.pem — dhparam;
ecss10.crt — публичный сертификат. Также содержит в себе указание на альтернативные доменные имена (Subject Alternative Name), используемые ECSS.

\\\Secure FTP\ecss_kms\etc\ecss\ssl\
##генерация ключа:
sudo dpkg-reconfigure ecss-user - (generate) создание сертификата на ecss1, предварительно удалить ecss10.{pem,crt,key}   и   ecss10root.crt
##копирование ключа, запускается с сервера на который нужно скопировать:
sudo gpasswd -a <USER> ssw - добавить пользователя, от имени котрого будет копироваться сертификат, в группу ssw.
sudo dpkg-reconfigure ecss-user - (copy) копирование сертификата с ecss1(запускается на ecss2), можно просто скопировать  ecss10.{pem,crt,key}   и   ecss10roo.crt
